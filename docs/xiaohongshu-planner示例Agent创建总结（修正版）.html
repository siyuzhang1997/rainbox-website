📋 三层记忆架构

第一层：静态配置文件（手动维护）

  • IDENTITY.md、SOUL.md、AGENTS.md、USER.md、TOOLS.md
  • 由 rainsyzhang 手动编辑
  • 更新频率：很少或偶尔

第二层：动态核心记忆（自动更新）

  • MEMORY.md：策划经验、爆款模式、平台洞察
  • 由 Agent 每次策划后自动更新
  • 包含元数据：last_used、count、type

第三层：日志归档（自动压缩）

  • memory/YYYY-MM-DD.md：每日策划日志
  • memory/self-improvement.md：失败教训
  • memory/archive/：归档历史日志
  • 定时任务自动压缩（每日 19:00）

💡 关键修正点总结

1. 记忆系统完善 ⭐

修正前：没有 MEMORY.md 和 memory/ 目录 修正后：完整的三层记忆架构

价值：

  • Agent 可以自动学习和积累策划经验
  • 每次策划都会更新核心记忆
  • 失败教训会记录到 self-improvement.md

2. readOnly 配置修正 ⭐

修正前：readOnly: true（策划不能写文件） 修正后：readOnly: false（策划需要写日志）

原因：

  • 策划需要记录日志到 memory/YYYY-MM-DD.md
  • 策划需要更新 MEMORY.md
  • 策划需要记录失败到 self-improvement.md

3. allowedTools 补充 ⭐

修正前：没有 write_to_file 修正后：增加 write_to_file

原因：

  • 配合 readOnly: false
  • 允许策划写日志

4. 删除 BOOTSTRAP.md

修正前：有 BOOTSTRAP.md（首次引导） 修正后：删除（标准结构没有）

原因：

  • 现有 Agent（xiaohongshu-writer、fullstack-developer）都没有 BOOTSTRAP.md
  • 首次引导应该通过其他方式实现

5. customRules 补充

修正前：只有职责边界规则 修正后：增加记忆更新规则

新增规则： 

"每次工作前，先读取 MEMORY.md 回顾策划经验",
