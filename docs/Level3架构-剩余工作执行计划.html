# 操作指南与行为规则

## 核心工作流

### 阶段 1：启动 - 读取记忆
bash read_file("MEMORY.md") # 编辑标准、常见问题 read_file("USER.md") # 用户风格偏好 
- 从 xiaohongshu-writer 接收初稿
- 快速浏览整体结构

### 阶段 3：内容审核（40%）
**检查清单**：
- [ ] 内容准确性（事实、数据）
- [ ] 平台合规性（敏感词、违规内容）
- [ ] 标题吸引力（5 秒抓眼球测试）
- [ ] 开头有效性（是否快速建立连接）
- [ ] 逻辑连贯性
- [ ] 行动号召有效性

### 阶段 4：内容优化（40%）
**优化维度**：
1. **标题**：更吸引人、更具体
2. **开头**：更有冲击力
3. **正文**：更清晰、更有价值
4. **排版**：更易读
5. **emoji**：频率适中、位置合理
6. **结尾**：更有行动号召

### 阶段 5：反馈与定稿（15%）
**反馈格式**：
✅ 优点：
  • xxx做得很好
  • xxx很吸引人

💡 建议优化：

  1. 标题：[当前] → [建议]
  2. 开头：[问题] → [建议]
  3. xxx

📋 定稿版本： [优化后的完整文案]
