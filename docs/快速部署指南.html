RainBox 官网快速部署指南

目标：10 分钟内完成 RainBox 官网部署到 GitHub Pages

🚀 方式一：一键部署（推荐）

使用自动化脚本

# 1. 运行部署脚本
./deploy-setup.sh

# 2. 按提示输入信息
# - 仓库名称（默认：rainbox-official-website）
# - 仓库描述（默认：RainBox 数字科技官网）
# - GitHub 用户名

# 3. 等待脚本完成（约 5 分钟）

# 4. 配置 GitHub Pages
# 访问：https://github.com/YOUR_USERNAME/rainbox-official-website/settings/pages
# Source 选择：GitHub Actions

# 完成！🎉

📝 方式二：手动部署

Step 1: 创建 GitHub 仓库（2分钟）

# 使用 GitHub CLI
gh repo create rainbox-official-website --public --description "RainBox 数字科技官网"

# 或手动创建
# 访问 https://github.com/new

Step 2: 初始化项目（3分钟）

# 创建 React 项目
npm create vite@latest rainbox-official-website -- --template react-ts

# 进入目录
cd rainbox-official-website

# 安装依赖
npm install

Step 3: 配置 Vite（1分钟）

编辑 vite.config.ts：

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  base: '/rainbox-official-website/', // 重要！
})

Step 4: 配置 GitHub Actions（2分钟）

创建 .github/workflows/deploy.yml：

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - uses: actions/deploy-pages@v4

Step 5: 推送代码（1分钟）

git init
git add .
git commit -m "feat: 初始化 RainBox 官网"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/rainbox-official-website.git
git push -u origin main

Step 6: 配置 GitHub Pages（1分钟）

  1. 访问：https://github.com/YOUR_USERNAME/rainbox-official-website/settings/pages
  2. Source 选择：GitHub Actions
  3. 保存

完成！🎉

访问：https://YOUR_USERNAME.github.io/rainbox-official-website/

🎨 自定义域名（可选）

Step 1: 购买域名

推荐平台：

  • 阿里云：https://wanwang.aliyun.com
  • 腾讯云：https://dnspod.cloud.tencent.com
  • Cloudflare：https://www.cloudflare.com

Step 2: 配置 DNS

添加 CNAME 记录：

主机记录: www
记录值: YOUR_USERNAME.github.io
TTL: 600

Step 3: 配置 GitHub Pages

  1. 访问：https://github.com/YOUR_USERNAME/rainbox-official-website/settings/pages
  2. Custom domain 填入：www.rainbox.tech
  3. 勾选 Enforce HTTPS
  4. 保存

Step 4: 等待生效（10-30分钟）

访问：https://www.rainbox.tech

🚀 Vercel 备用方案（可选）

为什么需要备用？

  • GitHub Pages 访问慢时切换
  • 双保险，高可用
  • 零额外成本

部署到 Vercel

# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 生产部署
vercel --prod

配置自定义域名

  1. 访问 Vercel Dashboard
  2. 选择项目 → Settings → Domains
  3. 添加域名：www.rainbox.tech
  4. 配置 DNS（Vercel 会提供指引）

🐛 故障排查

问题 1：GitHub Actions 构建失败

原因：依赖安装失败或构建错误

解决： 

# 本地测试构建
npm run build

# 查看 GitHub Actions 日志
# 访问：https://github.com/YOUR_USERNAME/rainbox-official-website/actions

问题 2：页面 404

原因：base 路径配置错误

解决： 

// vite.config.ts
export default defineConfig({
  base: '/rainbox-official-website/', // 必须与仓库名一致
})

问题 3：样式丢失

原因：资源路径错误

解决： 

// 使用相对路径


// 或使用 import
import logo from './assets/logo.png'

问题 4：GitHub Pages 未启用

原因：未配置 GitHub Pages

解决：

  1. 访问：Settings → Pages
  2. Source 选择：GitHub Actions
  3. 保存
