方法 2：使用脚本安装

创建安装脚本：

cat > /tmp/install-rainbox.sh << 'EOF'
#!/bin/bash

# RainBox Agent 安装脚本

echo "🚀 开始安装 RainBox Agent..."

# 1. 创建目标目录
echo "📁 创建目标目录..."
mkdir -p ~/Library/Application\ Support/Box/engine/agents/user/rainbox

# 2. 复制配置文件
echo "📋 复制配置文件..."
cp -r /Users/rainsyzhang/Box/output/519083a2-5f89-4826-b41f-a896277edb8e/rainbox-agent/* \
   ~/Library/Application\ Support/Box/engine/agents/user/rainbox/

# 3. 创建工作空间
echo "🏗️ 创建工作空间..."
mkdir -p /Users/rainsyzhang/Box/agents/rainbox/{memory,projects,docs,scripts}

# 4. 创建软链接
echo "🔗 创建软链接..."
ln -sf ~/Library/Application\ Support/Box/engine/agents/user/rainbox \
   /Users/rainsyzhang/Box/agents/rainbox/config

# 5. 验证安装
echo "✅ 验证安装..."
if [ -f ~/Library/Application\ Support/Box/engine/agents/user/rainbox/agent-config.json ]; then
    echo "✅ RainBox Agent 安装成功！"
    echo ""
    echo "📊 安装信息："
    echo "  配置目录: ~/Library/Application Support/Box/engine/agents/user/rainbox/"
    echo "  工作空间: /Users/rainsyzhang/Box/agents/rainbox/"
    echo ""
    echo "🎯 召唤方式："
    echo "  @rainbox"
    echo "  /mode rainbox"
else
    echo "❌ 安装失败，请检查源文件路径"
    exit 1
fi
EOF

# 执行安装脚本
chmod +x /tmp/install-rainbox.sh
/tmp/install-rainbox.sh

🔧 常见问题

Q1：安装后无法召唤 RainBox？

解决方案：

  1. 检查配置文件是否在正确的目录
  2. 重启 Box 应用
  3. 检查 agent-config.json 格式是否正确

Q2：RainBox 的记忆系统不工作？

解决方案：

  1. 检查 memory/ 目录是否存在
  2. 检查文件权限是否正确
  3. 手动创建缺失的文件

Q3：如何更新 RainBox 的配置？

解决方案：

  1. 编辑配置文件（SOUL.md、AGENTS.md 等）
  2. 重新加载 Agent（重启 Box 或切换模式）

Q4：如何备份 RainBox 的记忆？

解决方案： 

# 备份整个配置目录
cp -r ~/Library/Application\ Support/Box/engine/agents/user/rainbox \
   ~/Library/Application\ Support/Box/engine/agents/user/rainbox.backup.$(date +%Y%m%d)
