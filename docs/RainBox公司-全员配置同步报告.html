💡 配置同步的意义

为什么要两边都有配置？

1. 系统稳定性

  • 系统配置目录：Engine 读取，确保 Agent 能正常运行
  • 工作目录：备份配置，防止系统配置丢失

2. 用户便利性

  • 工作目录在用户文件夹下，方便查看和管理
  • 不需要进入系统目录（~/Library/Application Support/）

3. 开发调试

  • 可以在工作目录查看配置
  • 可以对比两边配置是否一致
  • 便于发现配置问题

4. 组织规范

  • 统一标准：所有 Agent 配置结构一致
  • 易于管理：批量检查、批量更新
  • 专业形象：体现公司管理规范

🔄 配置管理最佳实践

未来创建新 Agent 时

标准流程

  1. 创建配置文件
    ```bash
    # 在系统配置目录创建
    mkdir -p ~/Library/Application\ Support/Box/engine/agents/user/{new-agent}
    cd ~/Library/Application\ Support/Box/engine/agents/user/{new-agent}

# 创建所有必需文件
touch IDENTITY.md SOUL.md AGENTS.md USER.md TOOLS.md
touch MEMORY.md MEMORY_README.md agent-config.json
mkdir -p memory/archive
touch memory/self-improvement.md
```

  1. 立即同步到工作目录
    bash # 同步配置 cp -r ~/Library/Application\ Support/Box/engine/agents/user/{new-agent}/* \ /Users/rainsyzhang/Box/agents/{new-agent}/

  2. 验证配置完整性
    bash # 检查文件数量 ls ~/Library/Application\ Support/Box/engine/agents/user/{new-agent}/*.md | wc -l ls /Users/rainsyzhang/Box/agents/{new-agent}/*.md | wc -l # 两个数字应该相同（7）

配置更新时

原则：同时更新两边

# 1. 更新系统配置
vim ~/Library/Application\ Support/Box/engine/agents/user/{agent}/AGENTS.md

# 2. 立即同步到工作目录
cp ~/Library/Application\ Support/Box/engine/agents/user/{agent}/AGENTS.md \
   /Users/rainsyzhang/Box/agents/{agent}/AGENTS.md

定期检查

建议频率：每月1日（与三层记忆架构检查同时进行）

检查命令：

# 检查所有 Agent 配置同步状态
for agent in hr-manager fullstack-developer legal-advisor \
             xiaohongshu-planner xiaohongshu-writer xiaohongshu-editor \
             webnovel-planner webnovel-writer webnovel-editor \
             rainbox; do
  sys=$(ls ~/Library/Application\ Support/Box/engine/agents/user/$agent/*.md 2>/dev/null | wc -l)
  work=$(ls /Users/rainsyzhang/Box/agents/$agent/*.md 2>/dev/null | wc -l)
  if [ "$sys" = "$work" ] && [ "$sys" = "7" ]; then
    echo "✅ $agent"
  else
    echo "❌ $agent (需要同步)"
  fi
done

📝 经验总结

本次任务的教训

  1. 早期 Agent 遗留问题
    - hr-manager、fullstack-developer、legal-advisor 是最早创建的
    - 创建时流程不完善，没有同步到工作目录
    - 教训：创建流程必须标准化

  2. 新 Agent 创建也有遗漏
    - xiaohongshu-planner 是 Level 3 升级时创建的
    - 虽然流程规范了，但仍有遗漏
    - 教训：需要检查清单和验证步骤

  3. 配置同步的重要性
    - 不同步会导致用户无法查看配置
    - 可能影响 Agent 的正常工作
    - 教训：配置同步是基础设施，必须确保

改进措施

  1. 建立标准流程
    - 创建新 Agent 必须使用标准流程
    - 包含配置同步步骤
    - 包含验证检查步骤

  2. 定期检查机制
    - 每月1日执行配置同步检查
    - 由 HR Manager 负责执行
    - 发现问题立即修复

  3. 文档化
    - 将配置同步流程写入《RainBox公司管理规范》
    - 作为 Agent 创建的必须步骤
    - 作为月度检查的标准项目
