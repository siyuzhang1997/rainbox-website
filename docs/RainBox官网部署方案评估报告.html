🚀 方案一：GitHub Pages（推荐）

核心优势

  1. 完全免费
- 无服务器成本 - 无流量限制（合理使用） - 无存储费用

  1. 自动化部署
- GitHub Actions CI/CD - 推送代码自动构建 - 零运维负担

  1. 专业形象
- rainbox.github.io 或自定义域名 - 符合科技公司标准 - 开源社区认可

  1. 技术栈完美匹配
- React 静态站生成 - TypeScript 类型安全 - 现代化前端工程

实施步骤

Step 1: 创建 GitHub 仓库（5分钟）

# 1. 创建仓库
gh repo create rainbox-official-website --public --description "RainBox 数字科技官网"

# 2. 初始化项目
cd rainbox-official-website
git init
git remote add origin https://github.com/YOUR_USERNAME/rainbox-official-website.git

Step 2: 配置 GitHub Actions（10分钟）

创建 .github/workflows/deploy.yml：

name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Build
        run: npm run build
      
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4

Step 3: 配置 GitHub Pages（2分钟）

  1. 进入仓库 Settings → Pages
  2. Source 选择 "GitHub Actions"
  3. 保存配置

Step 4: 推送代码，自动部署（1分钟）

git add .
git commit -m "feat: 初始化 RainBox 官网"
git push origin main

部署完成！ 🎉

访问地址：https://YOUR_USERNAME.github.io/rainbox-official-website/

自定义域名配置（可选）

如果有域名（如 www.rainbox.tech）：

  1. 在域名 DNS 添加 CNAME 记录：
CNAME  www  YOUR_USERNAME.github.io

  1. 在仓库 Settings → Pages → Custom domain 填入域名
  1. 启用 HTTPS（自动）

技术要求

🏢 方案二：with.woa.com（腾讯内部）

核心优势

  1. 内网访问快
- 腾讯内部网络 - 低延迟

  1. 集成度高
- 可能与内部系统打通 - 统一管理

未知因素

由于我对 with.woa.com 平台不够了解，存在以下疑问：

  1. 是否支持静态站托管？
  2. 是否支持自定义域名？
  3. 是否支持自动化部署？
  4. 是否有流量/存储限制？
  5. 是否需要审批流程？

建议

如果 RainBox 官网主要面向腾讯内部用户，可以考虑此方案。但如果面向外部客户，GitHub Pages 更合适。

⚡ 方案三：Vercel（最佳替代方案）

核心优势

  1. 极速部署
- 推送代码后 30 秒上线 - 自动检测 React/Next.js - 零配置

  1. 全球边缘网络
- 比 GitHub Pages 更快 - 智能 CDN 分发 - 国内访问优秀

  1. 开发体验极佳
- 预览部署（每个 PR 独立预览） - 实时日志 - 性能分析

实施步骤

Step 1: 安装 Vercel CLI（1分钟）

npm install -g vercel

Step 2: 部署（1分钟）

cd rainbox-official-website
vercel

就这么简单！ 🚀

成本分析

| 项目 | 费用 | |------|------| | Vercel Hobby 计划 | ¥0 | | 自定义域名 | ¥50-100/年 | | 总计 | ¥0-100/年 |

限制

  • 个人版有带宽限制（100GB/月，足够小型官网）
  • 商业使用建议升级 Pro 版（$20/月）

🎯 最终推荐方案

推荐：GitHub Pages + Vercel 双保险

为什么？

  1. GitHub Pages 作为主力
- 代码托管在 GitHub（版本控制） - 自动化部署 - 完全免费

  1. Vercel 作为备用
- 如果 GitHub Pages 访问慢，切换到 Vercel - 两个平台都监听同一个 GitHub 仓库 - 零额外成本

  1. 灵活切换
- 通过 DNS CNAME 切换 - 无需修改代码 - 高可用性
