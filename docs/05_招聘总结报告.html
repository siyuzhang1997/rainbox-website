## 🎯 核心设计原则

### 1. 职责边界必须非常硬 ⭐⭐⭐

**为什么边界要这么硬？**
> 一旦角色边界模糊，AI 就会"自我和解"——写手写完觉得还行，编辑看了也觉得还行，最后质量就下去了。

**三个角色的硬边界**：
- **策划（Planner）**：只做大纲，❌ **不写正文**
- **写手（Writer）**：只写文章，❌ **不自己审核**
- **编辑（Editor）**：只审核，❌ **不直接改文章**

### 2. 三个角色的协作流程

```
策划 → 写手 → 编辑 → 写手 ↔ 编辑（循环，最多 3 轮）
  ↓       ↓       ↓           ↓
 大纲   初稿   分析报告   定稿修改
```

### 3. 编排器的作用

**编排器负责**：
- 任务分配和调度
- Gate 检查（质量把关）
- 产出传递
- 流程控制

**为什么需要编排器？**
- 确保流程严格执行
- 避免角色越界
- 统一质量标准
- 提升协作效率

### 4. 检查清单体系

**小红书业务线**：
- 通用检查清单（15 项）
- 小红书检查清单（40 项）
- 总计 55 项

**网文业务线**：
- 通用检查清单（15 项）
- 网文检查清单（54 项）
- 总计 69 项

**作用**：
- 标准化审核流程
- 确保质量稳定
- 避免遗漏问题

### 5. 控制态大纲

**长度**：约 400 行/篇（小红书）或 400 行/章（网文）

**包含**：
- 小红书：12 维度控制信息
- 网文：剧情点、人物行动、对话要点、悬念设计、爽点安排

**作用**：
- 让写手能直接动笔
- 确保内容方向正确
- 提升创作效率
