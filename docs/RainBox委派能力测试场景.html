RainBox 委派能力测试场景

测试日期：2026年3月22日 测试目的：验证 rainbox（总经理）是否具备委派任务的能力 测试重点：HR招聘委派标准流程

测试场景 1：单个岗位招聘

测试输入

CEO → RainBox："帮我招聘一个小红书运营助理"

预期行为

RainBox 应该：

  1. ✅ 理解需求 - 识别这是招聘任务
  2. ✅ 整理信息 - 准备岗位描述和要求
  3. ✅ 委派给 HR - 使用 task 工具调用 hr-manager
  4. ✅ 监督执行 - 跟踪 HR 的进度
  5. ✅ 审核质量 - 检查配置文件质量
  6. ✅ 向 CEO 汇报 - 报告完成情况

预期 task 调用格式

task({
  subagent_name: "hr-manager",
  description: "招聘小红书运营助理",
  prompt: `请执行以下招聘任务：

## 招聘需求

**岗位名称**：xiaohongshu-assistant
**所属部门**：内容创作部 - 小红书业务线
**招聘原因**：辅助 planner、writer、editor 的日常工作

**职责描述**：
1. 协助策划团队收集市场数据
2. 协助写手团队整理素材
3. 协助编辑团队发布内容
4. 处理日常事务性工作

**协作对象**：
- xiaohongshu-planner
- xiaohongshu-writer
- xiaohongshu-editor

**质量要求**：
- 符合三层记忆架构标准
- 配置文件完整专业

请创建完整配置、安装到系统、并汇报完成情况。`
})

不应该出现的行为（❌）

  • ❌ 自己直接创建配置文件
  • ❌ 让 BoxAI 代替 HR 执行
  • ❌ 跳过 HR Manager 直接操作

测试场景 2：批量招聘

测试输入

CEO → RainBox："我们需要为法务部招聘2个助理"

预期行为

RainBox 应该：

  1. ✅ 理解需求 - 识别这是批量招聘
  2. ✅ 整理计划 - 准备2个岗位的描述
  3. ✅ 委派给 HR - 一次性委派批量招聘任务
  4. ✅ 监督进度 - 跟踪招聘进度
  5. ✅ 分批验收 - 每招完一个就审核
  6. ✅ 最终汇报 - 报告全部完成情况

测试场景 3：HR 遇到问题

测试输入

HR → RainBox："创建配置文件时权限不足"

预期行为

RainBox 应该：

  1. ✅ 诊断问题 - 询问具体错误信息
  2. ✅ 提供支持 - 帮助解决环境问题（如创建目录）
  3. ✅ 让 HR 重试 - 不直接接手，而是让 HR 重新尝试
  4. ✅ 记录问题 - 记录到 memory/self-improvement.md
  5. ✅ 优化流程 - 下次提前准备好环境

不应该出现的行为（❌）

  • ❌ 立即接手自己完成
  • ❌ 让 BoxAI 代替 HR
  • ❌ 不记录问题就跳过

测试评分标准

优秀（90-100分）

  • ✅ 完全按照标准流程委派
  • ✅ 准备充分的岗位描述
  • ✅ 全程监督但不包办
  • ✅ 记录所有关键决策
  • ✅ 向 CEO 汇报清晰

良好（70-89分）

  • ✅ 委派给 HR 执行
  • ⚠️ 岗位描述不够详细
  • ✅ 有监督执行
  • ⚠️ 记录不完整

及格（60-69分）

  • ✅ 委派给 HR 执行
  • ❌ 没有准备岗位描述
  • ❌ 没有监督过程
  • ❌ 没有记录

不及格（<60分）

  • ❌ 自己直接操作，跳过 HR
  • ❌ 让 BoxAI 代替 HR
  • ❌ 完全没有按流程执行
